- 版本: v2.0
- 日期: 2026-03-04
- 状态: 开发中

## 一、背景

JoyAgent集成方案因以下问题无法继续：
1. JoyAgent项目代码质量差，存在大量null处理bug
2. Java技术栈与Python技术栈混合增加维护成本
3. 架构过于复杂，不符合"简单架构"原则

---

## 二、当前架构

### 2.1 核心理念

**"Database + AI = 智能BI"**

保持简单：
- 一个Python后端
- 一个前端界面
- 一个AI对话接口

## 五、AI Agent 设计

### 5.1 核心组件

```python
# src/ai/agent.py

class PBBIAgent:
    """PB-BI AI Agent"""
    
    def __init__(self):
        self.llm_client = LLMClient()  # LLM客户端
        self.mcp_tools = MCPTools()    # MCP工具集
        self.context = {}              # 上下文
    
    async def chat(self, user_input: str) -> str:
        # 1. 理解用户意图
        intent = await self.understand_intent(user_input)
        
        # 2. 决定使用哪些工具
        tools = await self.select_tools(intent)
        
        # 3. 执行工具调用
        results = await self.execute_tools(tools)
        
        # 4. 生成回复
        response = await self.generate_response(results)
        
        return response
```

### 5.2 工具集成

已有的MCP工具可直接使用：
- `pbbi_get_dataflows` - 获取数据流列表
- `pbbi_get_dataflow` - 获取数据流详情
- `pbbi_query_data` - 查询数据
- `pbbi_get_schema` - 获取数据库结构
- `pbbi_get_dashboards` - 获取看板列表

### 5.3 对话流程

```
用户: "列出所有数据流"
  │
  ▼
Agent: 调用 pbbi_get_dataflows
  │
  ▼
返回: "当前系统中有3个数据流：
       1. test (本地数据)
       2. 设备 (明道云)
       3. test (明道云)"
```
